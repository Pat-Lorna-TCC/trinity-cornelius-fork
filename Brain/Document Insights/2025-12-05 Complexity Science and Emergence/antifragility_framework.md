## Core Framework

Antifragility characterizes systems that **benefit** from variability in environmental perturbations, going beyond resilience (returning to baseline) to actually becoming stronger from exposure to stressors.

**Definition**: System output response to input variability shows positive convexity - gains exceed losses from fluctuations.

## Three-Level Distinction

**1. Fragile**:
- Harmed by variability
- Prefers stability and control
- Breaks under stress
- Example: Optimized-for-efficiency supply chain

**2. Resilient**:
- Returns to previous state after disturbance
- Absorbs shocks
- Maintains baseline
- Example: Backup systems, redundancy

**3. Antifragile**:
- Becomes stronger from disturbance
- Benefits from variability
- Improves beyond baseline
- Example: Immune system, learning organizations

## Mechanisms of Antifragility

**Key Quote** (Dagstuhl Seminar 2024):
> "Antifragile systems benefit from exposure to uncertainty and disturbances, by learning from encounters with such difficulties."

**Design Elements** (Organizational context):
1. **Redundancy**: Buffer capacity for unexpected demands
2. **Loose Coupling**: Failure containment, experimentation space
3. **Modularity**: Independent component evolution
4. **Scalability**: Expand/contract as needed
5. **Option Value**: Preserve multiple pathways
6. **Barbell Strategy**: Safe core + experimental edge

**Process**: Design elements create context for self-organization and beneficial adaptation

## Paradigm Shift in Disruption Response

**Traditional Approach**:
- Minimize disruptions
- Predict and prevent
- Return to stability
- Risk aversion

**Antifragile Approach**:
- Leverage disruptions for value creation
- Controlled exposure to stressors
- Improve beyond previous state
- Strategic risk-taking

**Quote** (Project Management 2024):
> "Disruptions can actually help to create unexpected value."

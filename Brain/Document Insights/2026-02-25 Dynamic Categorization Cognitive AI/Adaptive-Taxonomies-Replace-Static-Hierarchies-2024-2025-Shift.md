# Adaptive Taxonomies Replace Static Hierarchies (2024-2025 Paradigm Shift)

## Core Insight

**Paradigm shift (2024-2025)**: From manually designed, static ontological hierarchies → automatically induced, dynamically evolving schemas. LLM-powered systems now construct and adapt taxonomies automatically, incorporating novel relations and entity types without retraining.

**Key Innovation**: **AdaKGC** (Ye et al., 2023, highly cited 2024-2025) enables dynamic schema evolution - models incorporate new categories on-the-fly.

---

## Evidence

**Major Developments**:

1. **LKD-KGC** (Sun et al., 2025)
   - Adaptive, embedding-based schema integration
   - Automatically extracts and merges equivalent entity types
   - Vector clustering + LLM-based deduplication
   - Schema alignment emerges dynamically

2. **AdaKGC** (Ye et al., 2023)
   - Addresses dynamic schema evolution challenge
   - Novel relations and entity types added without retraining
   - Continual taxonomy expansion

3. **LLM-empowered KG Construction** (ICAIS 2025)
   - Survey of shift from manual → automatic schema design
   - Automatically induced schemas from data

**Citation**: Multiple sources - "LLM-empowered knowledge graph construction: A survey," arXiv:2510.20345, ICAIS 2025; Sun et al. (2025), Ye et al. (2023)

---

## Why This Matters

**Static Taxonomies Fail**:
- Domains evolve (new concepts, relationships)
- Manual updates expensive, slow
- Knowledge engineers bottleneck
- Rigid hierarchies can't capture fluidity

**Adaptive Taxonomies Win**:
- Automatic schema induction from data
- Dynamic evolution as knowledge changes
- No retraining required
- Emergent organization

---

## Mechanism: How LLMs Enable Adaptation

**Traditional Approach**:
1. Domain experts manually design ontology
2. Categories fixed at design time
3. Updates require expert intervention
4. Rigid, expensive, slow

**LLM-Powered Approach**:
1. Analyze corpus → extract entities and relations
2. Cluster semantically similar entities
3. LLM deduplicates and labels clusters
4. Schema emerges from data
5. New data → new clusters → schema evolves

**Key Advantage**: Schema reflects actual usage patterns, not designer assumptions
